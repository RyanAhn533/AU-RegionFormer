# Label Quality Analysis: 한국인 감정 데이터셋

## 개요

AU-RegionFormer의 성능 ceiling(80.7%)이 모델이 아닌 **데이터 라벨 품질**에 기인한다는 가설을 검증하기 위해, AI Hub 원본 annotator 데이터와 연세대 외부 검증 데이터를 교차 분석한 결과를 정리한다.

---

## 1. 데이터 구조

### 1.1 원본 데이터 (AI Hub)

| 항목 | 내용 |
|------|------|
| 피험자 | 1,100명 (전문인 48%, 일반인 52%) |
| 감정 | 기쁨, 분노, 슬픔, 중립 + 당황, 불안, 상처 |
| 촬영 | 감정 유도 후 다양한 배경에서 촬영 (10가지 배경) |
| 라벨링 | annotator 3인 독립 판정 (감정 + bounding box) |
| 규모 | ~240K (해제된 4감정 기준) |

**피험자 유형:**
- 전문인: 배우/연기자 (감정 표현 훈련 경험)

  "annot_A": {"faceExp": "기쁨"}, // annotator A 판정
  "annot_B": {"faceExp": "기쁨"}, // annotator B 판정
  "annot_C": {"faceExp": "슬픔"}  // annotator C 판정 (불일치 예시)
}
```

## 4. 교차검증: 3개 독립 신호의 일관성

### 4.1 사용 신호

1. **annotator 3인 완전 일치** — 원본 전문 라벨러의 합의
2. **majority = uploader** — annotator 다수의견이 원래 라벨과 일치
3. **isProf (전문인/일반인)** — 배우의 표정이 더 명확한지

### 4.2 단변량 검정

| 신호 | noise율 (True) | noise율 (False) | chi² | p-value | Odds Ratio |
|------|---------------|----------------|------|---------|------------|
| 3인 일치 | 10.4% | 12.6% | 424 | 2.7e-94 | 0.772 |
| majority=uploader | 10.8% | 14.4% | 579 | 7.3e-128 | 0.718 |
| 전문인 | 10.7% | 12.6% | 215 | 9.5e-49 | — |

→ 3개 신호 모두 **p < 10⁻⁴⁰** 수준으로 통계적 유의. 방향도 일관 (True일수록 noise↓).

### 4.3 3중 교차 조합

| annotator | majority | 피험자 | noise율 | 판정 | n |
|-----------|----------|--------|---------|------|---|
| 3인일치 | =uploader | 전문인 | **8.7%** | LOW | 54,269 |
| 3인일치 | =uploader | 일반인 | 10.9% | MED | 63,498 |
| 3인일치 | ≠uploader | 전문인 | 10.8% | MED | 4,778 |
| 불일치 | =uploader | 전문인 | 12.5% | MED | 35,068 |
| 불일치 | =uploader | 일반인 | 12.0% | MED | 35,133 |
| 불일치 | ≠uploader | 전문인 | 12.4% | MED | 23,950 |
| 불일치 | ≠uploader | 일반인 | **15.7%** | HIGH | 26,209 |
| 3인일치 | ≠uploader | 일반인 | **21.5%** | HIGH | 4,583 |

**최고 신뢰 8.7% vs 최저 21.5%** — 2.5배 차이.

### 4.4 감정별 극단 비교

| 감정 | 최고 신뢰 noise | 최저 신뢰 noise | 차이 | Cohen's h |
|------|----------------|----------------|------|-----------|
| 기쁨 | 5.5% | 8.9% | 3.4%p | 0.076 |
| 분노 | 15.0% | 19.0% | 3.9%p | 0.052 |
| **슬픔** | **12.3%** | **22.0%** | **9.7%p** | **0.171** |
| 중립 | 6.7% | 6.5% | -0.3%p | 0.022 |

→ **슬픔에서 3개 신호의 결합 효과가 가장 크다** (9.7%p). AU-RegionFormer의 sad recall 54% 문제와 직결.

### 4.5 로지스틱 회귀 (메타데이터 예측력)

| 변수 | |coef| | 방향 |
|------|--------|------|
| majority≠uploader | 0.113 | noise↑ |
| annotator 3인일치 | 0.090 | noise↓ |
| 피험자 성별 | 0.053 | 남성 약간↑ |
| 프레임 번호 | 0.023 | 미미 |
| 나이 | 0.012 | 미미 |

**AUC = 0.553** — 메타데이터만으로는 noise 예측력 제한적. 이미지 내용(visual feature)이 핵심 변수.

## 5. 결론

### 5.1 확정된 사실

1. **데이터 라벨 noise 존재 확인** — 슬픔/분노 클래스에서 annotator 일치율 48-51%, 연세대 검증에서도 16-17% "아님" 판정
2. **3개 독립 신호의 일관성 확인** — annotator 합의, uploader 라벨 일치, 피험자 유형 모두 같은 방향으로 noise 예측 (모두 p < 10⁻⁴⁰)
3. **슬픔이 가장 noisy** — 모든 지표에서 슬픔이 최대 noise 감정. 모델의 sad recall 54%와 직결
4. **기쁨/중립은 깨끗** — noise 6-7% 수준, 모델도 F1 0.96-0.98
5. **연세대 평가의 신뢰성** — inter-rater 98.7% 일치. 객관적 신호 포함 확인

### 5.2 한계

1. 개별 효과 크기는 small (Cohen's h < 0.2)
2. 메타데이터 기반 AUC 0.55 → 메타데이터만으로는 불충분, visual feature 필요
3. 연세대 검증은 4감정만 — anxious, hurt, surprised는 커버되지 않음
4. 사진당 1명 평가 → 개별 판정의 신뢰도는 제한적 (단, 2인 교차 시 98.7% 일치)

### 5.3 AU-RegionFormer 학습 적용 방향

메타데이터 기반 noise prior + visual feature 기반 noise detection 결합이 필요.

**Phase 1 (메타데이터 기반):**
- 원본 JSON의 annotator 3인 라벨 → conditional noise probability 계산
- 413K 데이터에 hash 매칭으로 noise prior 부여
- sample weight = 1 - P(noise | annotator_agreement, majority_match, isProf)

**Phase 2 (모델 기반):**
- cleanlab: AU-RegionFormer 5-fold CV → predicted probability 기반 noise 탐지
- AUM: 학습 중 per-sample margin 기록 → mislabel 후보 식별
- Phase 1 prior와 교차 검증

**Phase 3 (학습 반영):**
- 확실한 noise (Phase 1 + Phase 2 모두 flag) → 제거
- 의심 수준 (한쪽만 flag) → sample weight 감소
- annotator 3인 라벨 → soft label (KL divergence loss)

## 9. 추가 분석: 숨겨진 신호들

### 9.1 같은 사람 내 감정 간 rejection 상관

같은 피험자가 여러 감정에서 촬영 → 한 감정에서 rejection 높은 사람이 다른 감정에서도 높은가?

| 감정 쌍 | 상관 r | 해석 |
|---------|--------|------|
| 분노 × 슬픔 | **0.340** | 부정 감정끼리 연동 — 표정이 애매한 사람은 둘 다 애매 |
| 기쁨 × 슬픔 | 0.207 | 약한 양의 상관 |
| 기쁨 × 분노 | 0.129 | 약한 양의 상관 |
| 슬픔 × 중립 | -0.081 | 거의 독립 |

→ **분노×슬픔 r=0.34는 주목할 만함.** "부정 감정 표현이 약한 사람"이라는 피험자 수준 특성이 존재. 피험자별 quality factor로 활용 가능.

### 9.2 세트 번호별 피로도

| 구간 | 평균 rejection |
|------|---------------|
| 초반 (1-30) | **12.3%** |
| 중반 (31-60) | 11.5% |
| 후반 (61-90) | **11.0%** |

세트 번호 vs rejection: **r=-0.633, p≈0**

→ **후반으로 갈수록 rejection 감소.** 피로로 인해 판단이 느슨해짐. **초반 세트의 판단이 더 엄격하고 신뢰할 수 있음.**

### 9.3 Annotator 불일치 Confusion Flow

```
슬픔 → 상처:   21,327건 ★★★ (최대)
분노 → 불안:   15,763건 ★★
슬픔 → 불안:   13,900건 ★★
슬픔 → 중립:   12,741건 ★★
분노 → 중립:    9,800건 ★
분노 → 상처:    8,912건 ★
분노 → 당황:    8,763건 ★
분노 → 슬픔:    8,060건
기쁨 → 중립:    2,807건
```

→ **슬픔↔상처↔불안이 핵심 혼동 삼각형.** AU-RegionFormer의 sad→hurt 32% 오분류와 정확히 일치.

### 9.4 유효하지 않은 신호 (제외 확정)
- 프레임 번호: rejection 10.4~12.2% — 유의한 차이 없음
- 배경/장소: rejection 10.6~12.3% — 유의한 차이 없음
- 피험자 나이: 차이 미미

## 10. 논리 흐름 정리

### 발견의 계층 구조

```
Layer 0: 원본 데이터 구조 파악
  └→ 전문인 48% + 일반인 52%, annotator 3인, 7감정

Layer 1: 라벨 품질 문제 확인
  ├→ annotator 일치: 기쁨 92% vs 슬픔 48%
  └→ 모델 성능과 직결: sad recall 54%, happy recall 98%

Layer 2: 연세대 검증 데이터 해석 교정
  ├→ is_selected=1 = "이 감정 아님" (앱 코드 확인)
  ├→ inter-rater 98.7% → 객관적 신호 확인
  └→ annotator 일치도와 동일 방향 상관 (chi2=424, p≈0)

Layer 3: 다중 신호 교차 검증
  ├→ 3개 독립 신호 일관성: 8.7% vs 21.5%
  ├→ 슬픔에서 최대 효과 (9.7%p)
  └→ 메타데이터 AUC=0.55 → visual feature 필요

Layer 4: 숨겨진 구조 발견
  ├→ 피험자 수준 특성: 분노×슬픔 r=0.34
  ├→ 피로도 효과: 초반 > 후반 (r=-0.63)
  ├→ 혼동 삼각형: 슬픔↔상처↔불안
  └→ 프레임/배경 → 유의하지 않음 (제외)

Layer 5 (다음): 모델 feature 기반 신호
  ├→ cleanlab: model confidence 기반 noise 탐지 (진행 중)
  ├→ prototype distance: feature space 이상치
  └→ kNN consistency: 이웃 라벨 불일치

Layer 6 (최종): 다중 신호 결합 방법론
  └→ 통계적 prior + model feature + human verification 융합
```

## 11. Layer 5 결과: cleanlab (모델 confidence 기반)

### cleanlab 탐지 결과

AU-RegionFormer v2 best로 413K 전체 predicted probability 추출 → cleanlab `get_label_quality_scores` + `find_label_issues` 적용.

| 항목 | 수치 |
|------|------|
| 총 label issues | 538 / 413,122 (0.1%) |
| high quality (>0.8) | 298,248 (72.2%) |
| medium (0.5-0.8) | 111,442 (27.0%) |
| low (<0.5) | 3,432 (0.8%) |

### 감정별 cleanlab quality

| 감정 | quality score | label issues | issue率 |
|------|--------------|-------------|---------|
| happy | **0.9078** | 10 | 0.0% |
| neutral | 0.8761 | 14 | 0.0% |
| angry | 0.8644 | 27 | 0.0% |
| surprised | 0.8312 | 105 | 0.2% |
| sad | 0.8246 | 41 | 0.1% |
| **hurt** | **0.7736** | **191** | **0.3%** |
| **anxious** | **0.7724** | **150** | **0.3%** |

→ **Layer 1~4의 발견과 완벽히 일치:**
- happy/neutral = 깨끗 (annotator 92%+ 일치, 연세대 rejection 6-7%)
- hurt/anxious = 가장 noisy (혼동 삼각형의 핵심)
- quality 순서가 annotator 일치율 순서와 동일

### Phase 1 × Phase 2 교차검증

| 조건 | 건수 | 해석 |
|------|------|------|
| Phase 1만 flag (metadata noise) | 103,183 | metadata 기준 의심 |
| Phase 2만 flag (cleanlab) | 434 | 모델이 혼동하는 샘플 |
| **둘 다 flag** | **104** | **최고 확신 noise** |
| 어느 쪽이든 flag | 103,721 | 넓은 의심 범위 |

→ **104건은 metadata도 의심 + 모델도 확신이 없는 샘플 = 확실한 오라벨 후보**
→ Phase 1은 넓게 (103K), Phase 2는 좁게 (538) 잡음 → 상호 보완적

### 핵심 인사이트

cleanlab이 0.1%만 flag한 이유: 모델 자체가 80% accuracy밖에 안 되므로, **모델이 틀린 건지 라벨이 틀린 건지 구분 못함.** 이건 cleanlab의 한계 — 약한 모델로는 noise를 잘 못 찾는다.

→ 이래서 **feature-based detection (prototype distance, kNN)이 필요.** 모델 prediction이 아닌 feature space 구조로 이상치를 잡아야 한다.

### kNN 불일치율 해석

kNN 불일치율 = "50개 이웃 중 다른 감정인 비율". 높을수록 feature space에서 다른 감정과 섞여 있음.

- **happy 0.155** — 이웃 50개 중 85%가 happy. feature space에서 명확히 분리됨
- **anxious 0.543** — 이웃 50개 중 **절반이 다른 감정**. feature space에서 완전히 혼재
- **hurt 0.522** — anxious와 비슷하게 혼재

→ **anxious와 hurt는 feature space에서 다른 감정과 구분이 안 되고 있다.** 이건 라벨 noise만의 문제가 아니라 **감정 자체가 feature level에서 겹치는 것.** 모델 구조적 한계이기도 함.

### 전체 신호 일관성 확인

```

### v5 결과: soft label + sample weight → 실패

### 전체 실험 인사이트

**인사이트 1: 아키텍처 개선의 한계 확인**
- v1→v2→v3: 구조 변경으로 +1.2%p가 최대. 추가 구조 개선의 ROI 거의 없음.

**인사이트 2: v3의 과적합 역설**
- regularization(EMA+distill+mixup)을 넣었지만 과적합이 v2보다 심해짐 (8.6%→15.7%).
- 학습이 느려져서 더 많은 epoch 필요 → train만 올라감.

**인사이트 3: 라벨 정제로는 성능 향상 불가**
- soft label → 학습 방향 상실 (v5 F1=0.54)
- noise 제거 → feature가 겹치는 건 해결 안 됨
- **라벨이 아닌 feature가 bottleneck.**

**인사이트 4: AU attention 분석의 핵심 발견**
- 감정 간 AU cosine similarity 0.97-0.99 → 모든 감정이 거의 같은 AU 패턴
- anxious↔hurt similarity 0.999 → feature level에서 사실상 동일
- **현재 모델은 AU 강도의 0.01-0.03 수준 미세 차이로만 감정 구분** → noise에 극히 취약

**인사이트 5: 80% ceiling의 구조적 원인**
```
표정 이미지 → MobileViTv2 feature → AU attention → 감정 분류
                                          ↑
                              여기서 모든 감정이 0.97+ 유사
                              → 분리 불가능한 감정 존재
                              → 80%가 이 데이터의 실질적 상한
```

**인사이트 6: 성능 향상을 위한 필요 조건**
1. **Feature 품질 향상**: face-pretrained backbone (ArcFace/AdaFace 계열)
2. **추가 정보원**: 멀티모달 (음성 톤, 생체신호, 문맥)
3. **또는**: "표정 단독 7감정 인식의 한계"를 인정하고, 이를 contribution으로 제시

### 논문 관점에서의 가치

| 발견 | 논문 contribution |

### 18.1 감정별 인식 구조

### 18.2 혼동의 비대칭성

| 방향 | 비율 | 역방향 | 해석 |
|------|------|--------|------|

| 슬픔→불안 | **7.7%** | 불안→슬픔 0% | 일방향 |

| 기쁨↔중립 | 1.6% | 1.6% | **유일한 대칭** |

→ **부정 감정은 "강→약" 방향으로 일방적으로 혼동.** 슬픔이 상처로, 분노가 불안으로 읽힘. 이는 한국 문화에서 **강한 부정 감정이 절제되어 표현되는 경향**과 일치 — 강한 감정(분노)이 약하게 표현되면 불안으로 읽히고, 슬픔이 내재화되면 상처로 읽힘.

### 18.6 종합: 한국인 감정 표현의 문화적 특성

1. **긍정 감정(기쁨)은 보편적** — 성별/나이/전문성 무관 96%+ 인식
2. **부정 감정은 "강→약" 일방향 혼동** — 문화적 감정 절제의 영향
3. **젊은 여성의 부정 감정이 가장 모호** — 20대 여성 슬픔 인식률 추정 ~63%
4. **자연스러운 표현 > 연기된 표현** — 배우보다 일반인의 인식률이 높음
5. **한국인은 "볼(cheek)" 중심 표정 인식** — 서양의 이마/입 중심과 대비

이러한 경향성은 AU-RegionFormer의 80% ceiling을 설명하는 동시에, **한국형 FER 시스템 설계 시 고려해야 할 문화 특성적 가이드라인**을 제공.
